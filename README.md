# PROG7313-ARRAY-VENGERS-POE
Programming Assignment

Our BudgetApp is an easy-to-use Android app that helps users manage categories, keep track of their daily spending, and create spending reports. By providing users with precise information about where their money is going, the app encourages improved financial practices.

Features :
Authentication of Users
Secure access via the login screen
The ability to register for a new account
Management of Categories
Make your own categories for expenses, such as groceries, transportation, and bills.
Effectively manage your spending Track your expenditures

Include costs with:
Description, Amount, and Date
Time of Start and End
Choosing a category
The ability to upload a picture

Category:
Make your own categories, such groceries, water, and lights.
Facilitates effective expense organisation

The application architecture:
Guarantees the division of responsibilities:
Activity-Based UI Layer
Logic Layer: Controllers and ViewModels
Data Layer: Models and Databases


Spending Statements
Create reports using the chosen time ranges.
View the total amount spent for each category.
aids in identifying patterns of expenditure 

View Expenses 
View all of the recorded costs
Use a date range to filter expenses.
Easy-to-use list view with category and quantity

App Screens 
Included in the app are the following screens:
The login screen
Construct a Category Screen
Include the Expense Screen
View the Expense Screen
Display of Spending Reports

Application was built with :
Android Developer
Java and Kotlin 
XML Designs

How to run the application:
Make a clone of the Repo for GitHub: https://github.com/ST10255824/PROG7313-ARRAY-VENGERS-POE.git

Open Android Studio for the project
Sync files with Gradle
Run the software on Emulator called BlueStacks 
Budget applications opens
Start using the application as you wish 
Click and fill in the information you want 
Submit all the information required 
Submit 

Project structure:
/app
 ├── activities       Screens (Login, Add Expense, Reports, etc.)
 ├── models           Data classes (Expense, Category)
 ├── database         SQLite/Room database logic
 ├── adapters          RecyclerView adapters
 └── res               Layouts, drawables, values

Use Case: 
A user logs in.
creates categories, such as bills and groceries.
Daily expenditures are added
Views a list of expenses
creates a report for spending analysis.
